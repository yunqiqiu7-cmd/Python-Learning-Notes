## 一、需要纠正的认识

### ⚠️ `index()` 找不到时不是返回 -1，而是**报错**

你说"失败它返回 -1"，这是错的。`list.index()` 和 `str.index()` 在找不到元素时会**抛出 `ValueError`**，而不是返回 -1：

```python
[1, 2, 3].index(99)   # ValueError: 99 is not in list
"hello".index("xyz")   # ValueError: substring not found
```

返回 -1 的是**字符串专属的 `str.find()`**。这两个很容易混淆，区别如下：

| 方法 | 找到时 | 找不到时 | 适用类型 |
|---|---|---|---|
| `.index()` | 返回位置 | **抛出 ValueError** | list、tuple、str |
| `.find()` | 返回位置 | **返回 -1** | 仅 str |

所以如果你不确定元素是否存在，用 `find()` 更安全（针对字符串），或者先用 `in` 判断再用 `index()`。

### ⚠️ `map` 和 `replace` 的参数顺序没有矛盾

你说感觉 `map` 是"目标类型在前、原始类型在后"，而 `replace` 是"原始在前、目标在后"，想找统一规律。这里有个理解偏差——`map` 和 `replace` 根本不是同一类操作，不存在统一的"原始/目标"顺序规律。

`map(function, iterable)` 的参数是"函数"和"数据"，不存在"原始→目标"的转换概念。`int` 出现在前面只是因为它是要应用的函数，不是"目标类型"：

```python
map(int, ["1", "2", "3"])    # 把 int 函数应用到每个元素
map(str, [1, 2, 3])          # 把 str 函数应用到每个元素
map(lambda x: x**2, [1,2,3]) # 把平方函数应用到每个元素
```

`str.replace(old, new)` 的参数则是"旧子串"和"新子串"，语义是"把 old 替换为 new"，这个顺序符合自然语言"replace A with B"的逻辑。

Python 中没有一条通用规则规定所有函数的参数顺序，每个函数按自己的语义设计参数。遇到不确定的，查文档即可。

## 二、核心问题解答

### 1. 方法（`obj.method()`）与内置函数（`func(obj)`）的区别

这是你反复在问的问题。Python 中存在两种调用风格，它们的区别不是"两种写法"，而是**本质上不同的东西**：

**内置函数（Built-in Function）** 是 Python 语言层面提供的通用工具，不属于任何特定类型，能作用于多种类型。调用形式是 `function(object)`。典型的有：`len()`、`sorted()`、`max()`、`min()`、`sum()`、`zip()`、`map()`、`enumerate()`、`type()`、`isinstance()`、`print()`、`input()`、`range()`、`abs()`、`round()`、`ord()`、`chr()`、`hex()`、`bin()`、`oct()`、`eval()` 等。

**方法（Method）** 是绑定在某个类型上的函数，只有该类型（及其子类型）的对象才能调用。调用形式是 `object.method()`。比如 `list.sort()`、`list.append()`、`str.find()`、`str.replace()`、`str.isalpha()`、`dict.items()` 等。

判断规则很简单：如果一个功能是**很多类型都需要的通用操作**（比如求长度、排序、求最大值），Python 就倾向于做成内置函数；如果一个功能是**某个类型独有的操作**（比如字符串替换、列表追加），就做成方法。

`sort` 和 `sorted` 之所以"看起来像两种写法"，实际上是两个不同的东西——`list.sort()` 是 list 类型独有的方法（原地排序，返回 None），`sorted()` 是内置函数（适用于任何可迭代对象，返回新列表）。它们的功能相似但行为不同，并不是"同一个东西的两种写法"。

`count` 也是一样：没有内置函数 `count()`，只有 `list.count()` 和 `str.count()`，它们分别是各自类型的方法。

### 2. `max`、`min`、`sum` 的输入输出

这三个都是内置函数，接受**任何可迭代对象**（列表、元组、集合、字典、生成器等），要求元素之间可比较（`max`/`min`）或可相加（`sum`）。

对于字典，直接传入时遍历的是**键**：

```python
d = {'a': 3, 'b': 1, 'c': 2}

max(d)           # 'c'  ——键的最大值（按字母序）
min(d)           # 'a'  ——键的最小值
sum(d.values())  # 6    ——值的求和（直接 sum(d) 会对键求和，键是字符串会报错）
max(d.values())  # 3    ——值的最大值
max(d, key=d.get)# 'a'  ——值最大的那个键
```

返回值类型就是元素本身的类型——如果元素是 `int`，返回 `int`；如果是 `str`，返回 `str`。`sum` 的返回值是累加结果的类型，且 `sum` 不支持字符串拼接（要拼接字符串用 `''.join()`）。

### 3. `ord` 函数与字符编码

`ord()` 接受**一个字符**（长度为 1 的字符串），返回它的 **Unicode 码点**（整数）。ASCII 是 Unicode 的子集（前 128 个），所以 ASCII 字符的 `ord` 值就是 ASCII 码。

合法输入是任何单个字符——数字字符、字母、标点、空格、转义字符、中文、emoji 全都可以：

```python
ord('A')    # 65
ord('a')    # 97
ord('0')    # 48（注意：字符 '0'，不是数字 0）
ord('\n')   # 10（换行符，是一个字符）
ord(' ')    # 32（空格）
ord('中')   # 20013
```

注意 `'\n'` 虽然写起来是两个符号，但它是**一个字符**（转义字符）。反函数是 `chr()`，把整数转回字符：`chr(65)` 得到 `'A'`。

常用 ASCII 范围记忆：`'0'`-`'9'` 是 48-57，`'A'`-`'Z'` 是 65-90，`'a'`-`'z'` 是 97-122。

### 4. 进制转换

流程是你说的那样，先确保是 `int`，再用转换函数：

```python
n = int(input())  # 字符串 → 整数
bin(n)   # '0b1010'  —— 二进制，前缀 0b
oct(n)   # '0o12'    —— 八进制，前缀 0o
hex(n)   # '0xa'     —— 十六进制，前缀 0x
```

三个函数返回的都是**字符串**（带前缀）。如果不想要前缀，用切片 `bin(n)[2:]` 或 `format(n, 'b')`。

反向转换（任意进制字符串 → 十进制整数）用 `int(string, base)`：

```python
int('1010', 2)   # 10
int('12', 8)     # 10
int('a', 16)     # 10
```

### 5. 01 序列计数 0 的方法

你用"总长度减去 sum"的思路完全正确且很巧妙。更直接的方法确实是 `count`：

```python
s = input()

# 方法 1：你的思路（适用于整数列表）
nums = list(map(int, s.split()))
zeros = len(nums) - sum(nums)

# 方法 2：直接 count（适用于字符串或列表）
zeros = s.split().count('0')       # 对分割后的列表 count
# 或者如果输入是连续的01串
zeros = s.count('0')               # 直接对字符串 count
```

两种方法各有优势：`sum` 方法在数据已经是整数列表时很自然，`count` 方法更直观。

### 6. `index()` 的完整参数

```python
list.index(value, start=0, stop=len(list))
str.index(substring, start=0, end=len(str))
```

可以指定搜索范围，返回的是**原序列中的绝对位置**（不是相对于 start 的偏移量），这一点你的理解是对的：

```python
s = "hello world hello"
s.index("hello", 1)    # 12（从位置1开始找，返回的是原串中的位置12）
```

### 7. 多行输入直到遇到终止符

Python 的 `input()` 没有 `end` 参数，不能直接设置终止符。你用 `while True` 是标准做法。不过有一个更简洁的写法，用 `iter()` 的双参数形式：

```python
# iter(callable, sentinel) —— 反复调用 callable，直到返回值等于 sentinel 时停止
words = list(iter(input, '0'))
sentence = ' '.join(words)
```

`iter(input, '0')` 的含义是：反复调用 `input()`，当返回值等于 `'0'` 时停止迭代。这是一个非常优雅的写法，比 `while True` + `break` 简洁很多。

### 8. `eval()` 函数

`eval()` 把字符串当作 Python 表达式来求值：

```python
eval("3 + 5")        # 8
eval("2 ** 10")       # 1024
eval("'hello'.upper()")  # 'HELLO'
```

它的实现原理是调用 Python 的解析器和编译器，把字符串解析成抽象语法树（AST），然后在当前（或指定的）命名空间中执行。对于做题来说当作黑盒用就行，但在实际开发中**极度危险**——用户可以通过输入恶意代码来执行任意操作，所以生产环境几乎不用 `eval`。做算法题或自己的小脚本随便用没关系。

## 三、字符串常用方法汇总

因为你提到希望列举字符串常用的内置方法，这里按功能分类整理。以下全部是方法（`str.method()` 形式）：

**查找与计数类**：`str.find(sub, start, end)` 返回子串首次出现的位置，找不到返回 -1；`str.rfind(sub, start, end)` 从右向左找；`str.index(sub, start, end)` 同 `find` 但找不到时抛 ValueError；`str.count(sub, start, end)` 返回子串出现次数。

**判断类**（全部返回布尔值）：`str.isalpha()` 是否全为字母，`str.isdigit()` 是否全为数字，`str.isalnum()` 是否全为字母或数字，`str.isspace()` 是否全为空白字符，`str.isupper()` 是否全大写，`str.islower()` 是否全小写，`str.startswith(prefix)` 是否以某前缀开头，`str.endswith(suffix)` 是否以某后缀结尾。

**转换类**（返回新字符串，不改变原串）：`str.upper()` 转大写，`str.lower()` 转小写，`str.title()` 首字母大写，`str.strip()` 去除两端空白，`str.lstrip()` / `str.rstrip()` 去除左/右端空白，`str.replace(old, new, count)` 替换（count 可选，限制替换次数）。

**分割与拼接类**：`str.split(sep)` 按分隔符拆成列表（默认按空白），`str.rsplit(sep)` 从右拆分，`str.join(iterable)` 用该字符串拼接可迭代对象中的元素，`str.partition(sep)` 按首次出现的分隔符拆成三元组 `(前, 分隔符, 后)`。

**格式化类**：`str.format(...)` 格式化填充，`str.zfill(width)` 左边补零到指定宽度，`str.center(width, fillchar)` 居中填充，`str.ljust(width)` / `str.rjust(width)` 左/右对齐填充。

这些方法都不会修改原字符串（字符串是不可变类型），而是返回一个新字符串。

## 四、本章涉及的内置函数速查

| 函数 | 输入 | 输出 | 说明 |
|---|---|---|---|
| `max(iterable)` | 可迭代对象 | 最大的元素 | 元素需可比较 |
| `min(iterable)` | 可迭代对象 | 最小的元素 | 元素需可比较 |

| `ord(char)` | 单个字符 | int（Unicode码点） | 反函数是 `chr()` |
| `chr(int)` | 整数 | 单个字符 | 反函数是 `ord()` |
| `bin(int)` | 整数 | str（如 `'0b1010'`） | 二进制 |
| `oct(int)` | 整数 | str（如 `'0o12'`） | 八进制 |
| `hex(int)` | 整数 | str（如 `'0xa'`） | 十六进制 |
| `sorted(iterable)` | 可迭代对象 | 新的 list | 不改变原对象 |
| `eval(string)` | 字符串 | 表达式的值 | 做题可用，生产慎用 |
| `len(obj)` | 序列/集合/字典 | int | 元素个数 |
| `abs(number)` | 数值 | 数值 | 绝对值 |
| `round(number, n)` | 数值 | 数值 | 四舍五入到 n 位小数 |

## 五、小结

这章的核心区分点是**内置函数 vs 方法**：内置函数（如 `sorted`、`len`、`max`）是通用的，用 `func(obj)` 调用；方法（如 `.sort()`、`.find()`、`.replace()`）是类型专属的，用 `obj.method()` 调用。两者不是同一个东西的两种写法，而是设计层面的不同选择。

需要牢记的纠正项：`index()` 找不到时**抛异常**而非返回 -1（返回 -1 的是 `str.find()`）；`map` 的参数顺序是 `map(函数, 数据)`，和 `replace(旧, 新)` 不存在可比性。
