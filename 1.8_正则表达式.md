# Python正则表达式学习笔记

## 一、正则表达式核心语法（分组与重复匹配）

### 1\.1 问题起源

用户疑问：正则中`\*`的语法、无`?`的分组区别，以及多段号码的`group`匹配展示。

### 1\.2 核心概念：分组与`\*`重复

#### 1\.2\.1 捕获组 `\(\.\.\.\)` 与非捕获组 `\(?:\.\.\.\)`

- **捕获组 **`\(\.\.\.\)`：会将匹配内容存入分组，可通过`group\(n\)`获取（`n≥1`为分组编号，`0`为整个匹配）

- **非捕获组 **`\(?:\.\.\.\)`：仅用于「打包」正则单元供`\*`等重复修饰，**不生成分组编号**（解决「无`?`则成捕获组」的问题）

- 关键区别：是否保留匹配内容到分组（`?:`的`?`是标记非捕获的核心，无`?`则为捕获组）

#### 1\.2\.2 `\*`重复修饰符的特性

`\*`表示**匹配前一个单元 0 次或多次**，但对分组的影响分两种：

- 若分组是`\(A\)\*`（捕获组）：**仅保留最后一次匹配的 A**（前面的匹配会被覆盖）

- 若分组是`\(?:A\)\*`（非捕获组）：仅用于循环匹配，不保留任何分组内容

### 1\.3 实战测试：四段号码的四种正则写法

测试文本：`111\-222\-333\-444abc`（目标：匹配开头的四段连号）

#### 写法 1：普通捕获组 \+ `\*`（`\(\-\\d\+\)\*`）

```Plaintext
import re

s = "111-222-333-444abc"

p = r"\d+(-\d+)\*"  # 捕获组(-\d+)被\*重复

m = re.match(p, s)

print("group(0) =", m.group(0))  # 整个匹配：111-222-333-444

print("group(1) =", m.group(1))  # 仅保留最后一次捕获：-444

\# 无group(2)及以后（仅1个捕获组）
```

**适用场景**：仅需获取最后一次重复的内容，或仅需整体匹配

#### 写法 2：非捕获组 \+ `\*`（`\(?:\-\\d\+\)\*`）【最优推荐】

```Plaintext
p = r"\d+(?:-\d+)\*"  # 非捕获组(?:-\d+)被\*重复

m = re.match(p, s)

print("group(0) =", m.group(0))  # 整个匹配：111-222-333-444

\# 无group(1)（非捕获组不生成编号）
```

**适用场景**：仅需整体匹配，无需保留重复单元的内容（最常用的「循环打包」写法）

#### 写法 3：固定结构（写死四段）

```Plaintext
p = r"\d+-\d+-\d+-\d+"  # 无分组，纯整体匹配

m = re.match(p, s)

print("group(0) =", m.group(0))  # 整个匹配：111-222-333-444
```

**适用场景**：目标字符串结构完全固定（如固定四段）

#### 写法 4：全捕获（每段单独分组）

```Plaintext
p = r"(\d+)-(\d+)-(\d+)-(\d+)"  # 4个独立捕获组

m = re.match(p, s)

print("group(0) =", m.group(0))  # 整个匹配：111-222-333-444

print("group(1) =", m.group(1))  # 第一段：111

print("group(2) =", m.group(2))  # 第二段：222

print("group(3) =", m.group(3))  # 第三段：333

print("group(4) =", m.group(4))  # 第四段：444
```

**适用场景**：需要单独获取每一段的内容

### 1\.4 核心结论（复习重点）

**加粗部分为必须记忆的核心**

1. `\(A\)\*`**（捕获组 \+**`\*`**）：仅保留最后一次匹配的 A**

2. `\(?:A\)\*`**（非捕获组 \+**`\*`**）：仅用于循环匹配，不占分组编号**

3. 需拆分每段内容 → 手动写多个独立捕获组（如`\(\\d\+\)\-\(\\d\+\)\-\.\.\.`）

4. 截取开头整串编号的最优写法：`\\d\+\(?:\-\\d\+\)\*`

## 二、Python 正则常见错误排查

### 2\.1 错误案例：`TypeError: \&\#39;builtin\_function\_or\_method\&\#39; object is not subscriptable`

#### 错误代码

```Plaintext
print(m.group\[0])  # 错误：用方括号\[]调用方法
```

#### 错误原因

`group`是**匹配对象的方法**，需用 \*\* 圆括号`\(\)`\*\* 调用（方括号`\[\]`用于列表 / 字典等可迭代对象的下标访问）

#### 正确写法

```Plaintext
print(m.group(0))  # 正确：用圆括号调用方法
```

#### 完整修正代码

```Plaintext
import re

s = "111-222-333-444abc"

m = re.match(r'\d+(?:-\d+)\*', s)

if m:  # 先判断是否匹配成功（避免None调用group报错）

&#x20;   print(m.group(0))  # 正确调用

else:

&#x20;   print("无匹配")
```

#### 扩展提示

- 若`re\.match`未匹配到内容，会返回`None`，直接调用`group\(\)`会报`AttributeError`，**必须先判断匹配对象是否存在**

- 区分`group\(\)`与`groups\(\)`：

    - `group\(n\)`：获取第`n`个分组（`n=0`为整个匹配）

    - `groups\(\)`：返回所有捕获组的元组（无捕获组则为空元组）

## 三、算法题实战：网址前缀匹配（Nowcoder 题）

### 3\.1 题目描述

#### 输入

一行字符串表示网址（如`https://www\.Nowcoder\.com`）

#### 输出

用`re\.match`在起始位置匹配`https://www`的**匹配区间（起始下标，结束下标）** ，区分大小写；无匹配则输出`\(0, 0\)`

#### 示例

输入：`https://www\.Nowcoder\.com` → 输出：`\(0, 11\)`

### 3\.2 解题思路

1. **匹配规则**：仅从字符串开头匹配`https://www`（区分大小写，无修饰符）

2. **关键方法**：`re\.match`的`span\(\)`方法 → 直接返回匹配区间的元组（左闭右开，如匹配 11 个字符则结束下标为 11）

3. **边界处理**：若未匹配到，返回`\(0, 0\)`

### 3\.3 正确代码

```Plaintext
import re

url = input().strip()  # 读取输入并去除首尾空格

\# 仅从开头匹配https://www，区分大小写

match\_obj = re.match(r'https://www', url)

if match\_obj:

&#x20;   print(match\_obj.span())  # 输出匹配区间

else:

&#x20;   print((0, 0))  # 无匹配时输出(0,0)
```

### 3\.4 代码解释

- `re\.match\(r\&\#39;https://www\&\#39;, url\)`：仅从 url 开头匹配目标前缀，无`re\.IGNORECASE`修饰符（符合区分大小写要求）

- `match\_obj\.span\(\)`：返回匹配的起始和结束下标（左闭右开，如`https://www`共 11 个字符，结束下标为 11）

- 输入示例中，匹配成功，故输出`\(0, 11\)`

### 3\.5 知识点总结

- `re\.match`**特性**：仅从字符串**最开头**匹配，若开头不匹配则直接返回`None`

- `span\(\)`**方法**：快速获取匹配区间，适用于需要返回位置的题目

- **边界处理**：必须考虑无匹配的情况，避免`None`调用方法报错

## 四、复习总结（快速回顾）

### 正则基础

- 捕获组`\(\.\.\.\)`：存内容，可通过`group\(n\)`获取

- 非捕获组`\(?:\.\.\.\)`：不存内容，仅打包供`\*`重复

- `\*`重复：捕获组仅保留最后一次，非捕获组仅循环

### Python 正则语法

- `group\(\)`必须用圆括号`\(\)`调用，不可用方括号`\[\]`

- 匹配对象可能为`None`，需先判断

### 实战技巧

- 截取开头整串编号：`\\d\+\(?:\-\\d\+\)\*`（最优）

- 匹配位置区间：用`span\(\)`方法
